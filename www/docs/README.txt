** Заголовок

Документация требует наличия заголовка 1ого уровня (# в синтаксисе MDX). Он не будет отображаться на страницах,
так как уже есть в блоке Scene, но обязателен для указания в файле документации, по нему работает поисковик.

Полный пример в купе с headermeta:

```
<headermeta
  ...
/>

# Button

```



** Версионирование документации компонентов

Если предполагается версионировать документацию конкретного компонента, то 
контент документации заворачивается в специальный компонент:

```
<versionselector versionPages={[]}>
<crutch /> (данный элемент - это хак парсера mdx для нормального отображения послежующих кастомных компонентов внутри versionselector) 

...документация...

</versionselector>

```

он позволяет реализовать переключение разных версий документации.

*** examples

автогрид, самостоятельно падбирает количество колонок. 

ВАЖНО!!! В тег examples чилдренами передаем либо мап массива возвращающий JSX, например:

```
<example>
  {STATES.map(({ title, code, ...rest }) => (
    <example key={title} exampleCodeJSX={CODE_STATED}>
      <card title={title}>
        <StatedCheckbox {...rest} />
      </card>
    </example>
  ))}
</example>

```

либо руками прописываем каждый екзампл:

```
<examples>
  <example>
    <card title='Example 1'>
      <Example1 />
    </card>
  </example>
  <example>
    <card title='Example 2'>
      <Example2 />
    </card>
  </example>
  ...
</examples>

```
